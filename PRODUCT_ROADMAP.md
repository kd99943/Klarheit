#### 数据库

**阶段验收测试：**

### Phase 2 — AR 虚拟试戴（已完成 ✅）

**目标：** 实现核心差异化功能，让用户在购买前能虚拟试戴眼镜。

**技术选型及完成清单 (方案 A: MediaPipe Face Landmarker + Three.js)：**

- [x] 集成 MediaPipe Face Landmarker 实现实时人脸 3D 关键点检测
- [x] 使用 Three.js 将 3D 眼镜模型与姿态矩阵叠加到视频流上
- [x] **高保真 3D 空间正交基对齐**：使用左眼、右眼、鼻梁、鼻尖 3D 关键点解算面部局部坐标系轴向（dirX, dirY, dirZ），实现与头姿（Yaw/Pitch/Roll）高达 90° 的完美同步对齐，彻底解决侧脸虚空悬浮与眼距缩放漂移问题。
- [x] **物理精准材质与多层减反射膜（AR Sheen）**：镜片深度倒角挤压结合 `MeshPhysicalMaterial` 的折射、反射和蓝绿防反射膜 sheens 质感，完美重现蔡司级玻璃透光效果。
- [x] **柔光摄影棚环境图（EnvMap）与 PCF 软阴影**：1024x512 高清柔光箱渐变反射贴图，搭配 PCF 滤波阴影（shadow.radius = 4）以及 0.22 软阴影接收球，使镜框呈现高级高光且在面部投射极软的自然阴影。
- [x] **Silhouette 级无框（Rimless）钛金镜架建模**：为 ORBIT T-4 采用无框设计，支持镜片边缘微型穿孔夹片（mounting brackets）、超细桥梁和圆润镜脚，解决金属块的塑料感和 CGI 感。
- [x] **交互体验升级：镜框系列折叠选择器（Scheme C）**：移除简陋的眼镜颜色选择器，重构为可点击收缩与展开（Collapsible）的产品系列横向滑动选择栏。选择栏包含镜架高清缩微图片（Thumbnails）与材质描述描述，点选后即可实时拉取 3D WebAR 材质，支持镜架库的平滑扩展。
- [x] 摄像头权限管理、动态加载及支持前后摄像头转换
- [x] 切换不同产品时，AR 3D 材质和颜色实时渲染更新
- [x] AR 页面添加截图/本地下载功能
- [x] 完善的降级处理机制（无摄像头、人脸丢失、浏览器不兼容等异常状态友好提示）

## 优先级总结

```
Phase 0（地基）→ Phase 1（流程闭环）→ Phase 2（AR 核心功能）
     → Phase 3（支付）→ Phase 4（体验提升）→ Phase 5（部署上线）
```

**关键路径：** Phase 0 和 Phase 1 是所有后续工作的前提，必须先完成。Phase 2 是核心差异化，应在支付之前完成，因为 AR 体验是用户选择 Klarheit 而非竞品的核心理由。
