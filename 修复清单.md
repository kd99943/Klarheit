 问题 1：支付回调签名验证

  PaymentCallbackController.java — 核心改动：
  - Alipay 回调：在处理支付结果前，先验证 HMAC-SHA256 签名。无签名或签名校验失败直接返回 400 failure
  - WeChat 回调：同样加了签名校验，优先用 WeChat API key，退而使用共享 callback secret
  - 新增 verifyHmacSignature()、verifyHmacSignatureXml()、hmacSha256() 三个工具方法
  - 减少了日志中的敏感数据输出

  配置文件新增 PAYMENT_CALLBACK_SECRET：
  - application.yml — 新增 app.payment.callback-secret 配置项
  - .env — 本地开发用的默认值
  - .env.example — 文档化说明
  - docker-compose.yml — 传递给 backend 容器

  ▎ ⚠️ 上线前必须把 PAYMENT_CALLBACK_SECRET 改成一个强随机值（openssl rand -hex 32），并对接支付宝 SDK 的
  ▎ AlipaySignature.rsaCheckV1 做真正的 RSA2 验签。

  问题 2：nginx CSP 配置

  nginx.conf — 三处修复：
  - 移除 unsafe-eval（你的代码不需要 eval，移除后 XSS 防护更强）
  - connect-src 中的 http://localhost:8080 改为 ${API_ORIGIN} 占位符，构建/运行时可配置
  - 移除已废弃的 X-XSS-Protection 头
  - 移除 img-src 中不相关的 https://example.com

  Dockerfile（前端） — 支持运行时配置：
  - 安装 gettext（提供 envsubst）
  - nginx.conf 作为模板，启动时用 envsubst 替换 ${API_ORIGIN}

  docker-compose.yml（前端服务） — 新增 API_ORIGIN 环境变量

  ▎ ⚠️ 生产部署时设置 API_ORIGIN=https://api.klarheit.com（你的实际后端域名），否则浏览器的 CSP 会拦截 API 请求。

  问题 3：AR虚拟试戴（WebAR Try-On）与3D模型与定位对齐优化

  ARTryOnCanvas.tsx / ARVirtualStudio.tsx — 核心改动：
  - 3D正交基姿态解算：摒弃了之前粗糙的2D屏幕距离估算算法（该算法容易导致侧脸或倾斜时眼镜姿态异常，且将头部偏航角度限制在±20度左右），通过左眼(33)、右眼(263)、鼻梁(168)和鼻尖(1)的3D坐标点构建空间正交基（dirX, dirY, dirZ）以生成THREE.Quaternion。眼镜和脸部遮挡球现在能够与头部旋转（最高达90度侧脸）保持完美同步。
  - 动态深度与视锥缩放：根据眼距自动估算面部与相机的3D物理距离，并结合实时的THREE.PerspectiveCamera长宽比（aspect）和视角（fov）对试戴大小及位置进行绝对对齐，消除了因窗口拉伸或移动产生的漂移问题。
  - 修复页面销毁崩溃：在 `activeModel.traverse` 中对 geometry 和 material 的释放逻辑中加入空值校验（if (material)），防止跳出 React 提交阶段导致的页面意外报错（ErrorBoundary）崩溃。
  - 极致逼真的 3D 材质渲染：
    - 镜片改为 ExtrudeGeometry 3D 挤压带倒角几何体，配合 MeshPhysicalMaterial 物理折射（transmission: 0.95, ior: 1.56），真实还原玻璃边缘高光。
    - 镜片注入 cyan 蓝绿色 Sheen 光泽，细腻模拟高级配镜的多层防反射减反射膜（AR Coating）反光质感。
    - 采用 1024x512 高清渐变 Canvas 作为环境贴图（envMap），采用径向与线性渐变绘制逼真的摄影棚柔光箱，让金属镜架散发高级的 specular 倒角流光。
    - 镜脚等部位由 BoxGeometry 升级为 Extrude 倒角圆角柱，防止 CGI 模型常见的坚硬塑料感。
  - 柔和面部投影：开启 `keyLight.shadow.radius = 4` 的 PCF 软阴影，将阴影接收球的不透明度降至 `0.22`，不仅去除了之前脸上生硬的阴影黑斑，而且为镜架投射了极具质感的柔和局部阴影。
  - 100% 无框（Rimless）Orbit T-4 镜架：设计无框镜架替代廉价金属大圆圈，根据 Silhouette The Refined 5593 款式重新建模，包含镜片直接穿孔打钉固定的微型金色金属夹片配件、超细桥梁和圆润镜脚。
  - 规范物理色彩代码：更换 flat 高饱和颜色，启用沙箱内精准的金黄（#dfc17f）、玫瑰金（#e0a899）与金属钛（#a0a5ad）等配色。
  - 交互体验升级（镜框系列折叠选择器）：在 `ARVirtualStudio.tsx` 中取消了单一且缺乏表现力的颜色切换选择，重构为带点击折叠/展开（Collapsible）功能的镜框产品系列选择器。横向滑动卡片展示每个镜架的高清缩略图（Thumbnails）与材质描述，使用户可直观快速点选，并瞬间同步加载 AR 3D 镜框材质。

  问题 4：用户资料中心、手机号安全绑定与密码重置系统

  数据库迁移与字段新增：
  - V9__deduplicate_products.sql — 数据合并：清理历史数据中的重复商品，将关联订单迁移合并到标准产品主键 ID。
  - V10__add_phone_to_users.sql — 结构迁移：在 `users` 表中新增 `phone` 和 `phone_verified` 字段，并创建唯一索引保证安全性。

  后端服务重构：
  - 新增 `com.klarheit.backend.sms` 包：
    - `SmsService` / `MockSmsService`：用于生成/模拟发送 6 位数短信验证码。
    - `VerificationCodeStore`：基于线程安全哈希表或外部存储的验证码过期校验库。
    - `VerificationCodeService`：处理验证码的派发与安全核销逻辑。
    - `PhoneController`：提供 `/api/v1/phone/send-code`、`/verify-and-bind` 和 `/status` 接口，实现手机号绑定。
  - 安全扩展：
    - `AuthController` 新增 `POST /api/v1/auth/forgot-password` 接口，整合短信服务校验码与密码哈希重置机制，实现安全免密找回。
    - 同步调整 JWT Cookie 的 `SameSite` 策略为 `Lax`，提升第三方回调跳回时的会话一致性。

  前端个人资料与找密界面：
  - `ProfileDetailsPage.tsx`：全新的个人资料看板，清晰展示用户名字、邮箱、右左眼配镜度数（SPH/CYL/AXIS）以及 PD 瞳距数据卡片，同时整合绑定状态。
  - `PhoneBindingSection.tsx`：个人资料页嵌入的快捷绑定子组件，支持即时发送短信并输入 6 位验证码校验，实现无缝绑定。
  - `ForgotPasswordFlow.tsx`：登录抽屉中新增的“忘记密码”重置流程。引导用户通过“输入手机号 -> 接收并输入 6 位验证码 -> 校验成功设定新密码”，完全打通免密安全恢复。
